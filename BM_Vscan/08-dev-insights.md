# 8. 개발 참고 시사점 (직접 앱을 만들 경우)

1. **핵심 UX 패턴**: 화면 자체는 최대한 비우고, 좌(프리셋/메뉴)·우(exam/갤러리) 스와이프 패널로 기능을 숨기는 미니멀 디자인. 패널 열면 자동 freeze.
2. **연결 상태의 시각화**가 매우 중요 (배터리/신호세기/프로브 온도를 상단에 상시 노출) — 무선 특성상 리뷰에서도 가장 많이 언급된 약점.
3. Exam 단위로 이미지/비디오를 그룹핑하고, exam 종료 시점에 일괄 export/DICOM 전송하는 워크플로우.
4. Guest mode(비로그인 긴급 사용)를 계정 기반 플로우와 병행 설계.
5. DICOMweb 같은 표준 프로토콜 지원이 "엔터프라이즈/PACS 연동" 요구사항의 핵심.
6. 실시간 원격 화면공유(Zoom SDK 활용 사례처럼 기존 화상회의 SDK를 임베드하는 방식)로 "교육/원격지원" 기능을 저비용으로 구현 가능.
